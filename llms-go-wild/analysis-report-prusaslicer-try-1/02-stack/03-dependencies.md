# PrusaSlicer Dependencies

## Required Dependencies

### Core Dependencies
- **Boost (1.83.0+)**: 
  - Components: system, filesystem, thread, log, locale, regex, chrono, atomic, date_time, iostreams, nowide
  - Purpose: General-purpose C++ libraries for system operations, file handling, threading, logging, and text processing

- **Eigen3 (3.3.7+)**:
  - Purpose: Linear algebra library for mathematical operations in 3D geometry and slicing algorithms

- **Intel TBB (Threading Building Blocks)**:
  - Purpose: Parallel processing and multithreading for performance optimization

- **CURL**:
  - Purpose: HTTP/HTTPS client functionality for network operations
  - Additional dependency: ZLIB (on non-Windows platforms)

- **OpenGL**:
  - Purpose: 3D rendering and visualization
  - Implementation: Legacy OpenGL (as per CMakeLists.txt)

- **GLEW (OpenGL Extension Wrangler)**:
  - Purpose: Managing OpenGL extensions across different platforms and implementations

- **EXPAT**:
  - Purpose: XML parsing for configuration files and AMF format support

- **PNG**:
  - Purpose: PNG image format support for layer output and UI elements

- **Cereal**:
  - Purpose: C++11 serialization library for saving/loading application state

- **NLopt (1.4+)**:
  - Purpose: Nonlinear optimization library for print parameter optimization

- **OpenVDB (5.0+)**:
  - Purpose: Volumetric data processing for advanced geometry operations
  - Additional dependencies: IlmBase::Half, Blosc::blosc (when using OpenVDB)

## Dependency Management Strategy

### Static vs Dynamic Linking
- **SLIC3R_STATIC** option:
  - When enabled: Links against static versions of Boost, TBB, and GLEW
  - Platform-specific defaults: Enabled for Windows and macOS, disabled for Linux

### Cross-Platform Considerations
- **CMAKE_PREFIX_PATH**: Used to specify custom dependency locations
- **Config scripts**: Preferred over find modules when building with static dependencies
- **System-specific flags**: Applied based on platform detection

## Version Management
- **version.inc**: Contains version information generated by build script
- **Git integration**: Version numbers generated from git current label
- **Build ID**: Includes version and potentially git hash information
